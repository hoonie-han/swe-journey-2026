## What I Learned Today

**Semantic Tags** — HTML tags that carry meaning

- `<header>` Top of page or section
- `<nav>` Navigation menu
- `<main>` Main content of the page (only one per page)
- `<section>` Thematic grouping of content
- `<article>` Independent, self-contained content (blog post, news article)
- `<footer>` Bottom of page or section
- vs `<div>` Generic box with no meaning (used for layout)

These semantic tags placed in `<body>`

<html>
<head> → settings (invisible)
<body> → visible content

**How to Apply CSS**

**3 ways, but use only #3 in practice.**

1. **Inline** — `<h1 style="...">` — Avoid (only for JS or emails)
2. **Internal** — `<style>` in `<head>` — Only for small demos
3. **External** — `<link rel="stylesheet" href="style.css">` ← **Standard**

**##**

**Why external?**

- Separation of concerns (HTML = meaning, CSS = looks)
- Reusable across many pages
- Easy to maintain (change one file, updates everywhere)
- Browser caching → faster load

## Important

HTML은 "의미"를 정하고, CSS는 "보이는 모습"을 정한다.

**Core principle:** *Separation of concerns*

- HTML defines **what something is** (meaning)
- CSS defines **how it looks** (appearance)

---

### Choose tags by meaning, not by how they look

| Tag | Meaning |
| --- | --- |
| `<h1>` – `<h6>` | Heading hierarchy (NOT font size) |
| `<strong>` | Important content (looks bold) |
| `<em>` | Emphasized content (looks italic) |
| `<p>` | Paragraph |
| `<ul>` | Unordered list (order doesn't matter) |
| `<ol>` | Ordered list (order matters) |
| `<header>` | Top section of page or article |
| `<main>` | Main content |
| `<footer>` | Bottom section |
