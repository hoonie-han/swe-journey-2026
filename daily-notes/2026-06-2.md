Box Model

Padding & Margin — Individual Sides
Set each side separately
padding-top / padding-right / padding-bottom / padding-left
margin-top / margin-right / margin-bottom / margin-left
Shorthand (clockwise: top → right → bottom → left)
margin: 10px 20px 30px 40px;
= top 10, right 20, bottom 30, left 40
Order tip: start at 12 o'clock, go clockwise (top-right-bottom-left)
By number of values
`margin: 10px;`              → all 4 sides 10
`margin: 10px 20px;`         → top/bottom 10, left/right 20
`margin: 10px 20px 30px 40px;` → top, right, bottom, left
Tip
span is inline → width/height won't work
→ use display: inline-block to enable sizing

`box-sizing` controls how `width` and `height` are calculated.

`content-box` is the default.

`width` and `height` only include the content area.

Padding and border are added outside, so the actual box becomes bigger.

`border-box` includes content, padding, and border inside the set `width` and `height`.

So the actual box size stays the same.

Example:

`width: 100px`

`padding: 10px`

`border: 3px`

With `content-box`, actual width = `126px`.

With `border-box`, actual width = `100px`.

Common practice:

- `{ box-sizing: border-box; }`

This makes layout easier and more predictable.
