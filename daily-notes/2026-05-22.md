**Container Tag**

A **container** is a tag that doesn't change how content looks or where it sits on the page. Its only job is to wrap several other elements together so they're easier to manage.

- <div></div>: block level container
- <span></span>: inline container

You use a container when you want to:

- **group** a chunk of content as one unit, or
- **apply the same style** to everything inside it at once.

<div style="background-color: lightyellow; padding: 20px; color: navy;">
<h2>Announcement</h2>
<p>first sentence.</p>
<p>second sentence.</p>
</div>

**HTML Global Attribute**

**`id`** — A unique name for **one** element.
→ Like a social security number. Only one element per page can have it.
→ Used to point to *that one specific* element with CSS (`#name`) or JavaScript.

**`class`** — A label you can put on **many** elements.
→ Like Shift-clicking several files and styling them all at once.
→ Once tagged, they stay grouped — change the CSS one time, every element with that class updates.
→ Used in CSS as `.name`.

**`style`** — Writes CSS directly on the element itself.
→ Quick one-off styling, but messy if overused. Prefer `class` + a CSS file.

**`title`** — Extra info shown as a tooltip when you hover over the element.
→ Not for styling or selecting — just a little popup hint for the user.
