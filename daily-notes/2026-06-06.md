# Day 12 (2026-06-06)

## What I Learned Today

### HTML Review

- When starting HTML, first plan out what structure you'll use (with a sketch or design tool)
- Don't overuse `<div>` — use meaningful tags
  - ex. `<header>` `<nav>` `<main>` `<article>` `<section>` `<aside>` `<footer>`
- Getting the big picture first is important

### Element vs Container

- **Element** (holds content directly): `h1`~`h6`, `address`, `blockquote`, `p`, `pre`, `table`, `ol`, `ul`, `a`, `abbr`, `audio`, `b`, `span`, `cite`, `code`, `data`, `i`, `mark`
- **Container** (wraps other elements): `div`, `section`, `article`, `header`, `footer`, `aside`, `nav`, `main`

### Block vs Inline

- **Block** (takes full width, stacks vertically): `h1`~`h6`, `address`, `blockquote`, `p`, `pre`, `table`, `ol`, `ul`
- **Inline** (sits in a line, only as wide as content): `a`, `abbr`, `audio`, `b`, `span`, `cite`, `code`, `data`, `i`, `mark`

### See How Other Sites Are Built

- Open DevTools (`F12`) and study how other websites are structured.
